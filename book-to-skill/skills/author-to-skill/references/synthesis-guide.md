# Author synthesis guide

This is the work a script *cannot* do and single-book distillation never reaches:
turning N book maps into one coherent picture of the author's mind. Read this
before writing the Tier-0 map.

## The core move: semantic framework clustering

The same idea recurs across an author's books under different names and framings.
Your job is to recognize those as one thing and produce a **canonical version**,
while recording how each book frames it.

**Cluster by function, not wording.** Two passages describe the same framework if
they play the same *role* in the author's thinking — same problem solved, same
inputs, same move — even with zero shared vocabulary. The test: *would the author
consider these the same idea, or two different ones?* Decide semantically; never
cluster by keyword overlap.

For each cluster:
1. **Canonical name** — prefer the author's own most-used or most-developed term.
2. **Canonical gloss** — the version that holds across all appearances.
3. **Per-book variants** — where a book renames, extends, narrows, or complicates
   it ("Book C reframes this as X and adds the Y step").
4. **Depth pointer** — which book map + chapter toolkit to load for the full treatment.

**Qualification bar for Tier 0.** A framework belongs in the always-loaded author
map only if it (a) recurs across **≥2 books**, or (b) is a single-book idea that is
plainly a pillar of the author's whole project. Everything else stays in its book's
map — the author layer is the *shared* skeleton, not a merge of all content.

## The other four synthesis outputs

- **Through-lines / worldview** — the themes, commitments, or questions the author
  returns to. If the body of work is genuinely eclectic, say so honestly ("no single
  worldview; the constant is a *method* / *temperament*") rather than forcing a
  false unity.
- **Evolution timeline** — in publication order: which book introduced an idea,
  which refined it, where the author **reversed** an earlier position. This section
  lives or dies on publication years; without them, weaken it or drop it — do not
  invent an order.
- **Method & voice** — how the author argues: preferred evidence (stories, data,
  history, first-person), recurring rhetorical moves, how chapters are structured,
  characteristic stance. This is orthogonal to content and is the most useful part
  for actually *thinking like* the author. Distill it explicitly.
- **Bibliography graph** — per book: what it's for, when to read it, what it depends
  on or pairs with; plus a suggested reading order.

## Two-pass method

1. **Pass 1 — per book, standalone.** Run book-to-skill on each EPUB independently.
   Each book gets an honest map + toolkits with no knowledge of the others.
2. **Pass 2 — reconcile.** After the author map exists, revisit each book map and
   add back-references to the canonical author-layer frameworks, so five books don't
   each re-explain the same model. A book map keeps only *its* variant plus a pointer
   up ("Framework X: see author map for the canonical form; this book adds Y").

## Honest-handling pitfalls

- **False merge** — two ideas share a name but differ in substance; keep them
  separate. Reason about substance, not labels.
- **Missed merge** — the same idea wears different words in each book; catch it by
  reasoning about function. This is why clustering must be semantic.
- **Forced coherence** — don't manufacture a grand unified theory for an author who
  doesn't have one. Eclecticism reported honestly is more useful than a tidy lie.
- **Evolution without dates** — no reliable years → no confident timeline.
- **Co-authored / ghostwritten volumes** — dilute the "author" signal; note them,
  weight them less, and consider excluding a book with a different true author.
- **Later revises earlier** — mature books often correct early positions. The author
  map should reflect the author's **current** view *and* preserve the evolution arc
  (what changed and when), not silently overwrite the past.
- **Copyright** — synthesize and paraphrase. Never reproduce long verbatim passages;
  canonical glosses and variants are in your own words.
