# 打造第二大脑 — Working Knowledge

> 蒸馏自 Tiago Forte《Building a Second Brain》。这里是可复用的思维骨架,不是内容摘要。
> 需要某个环节的操作细节时,加载 `references/` 里对应的章节工具箱。

## Thesis
知识工作的瓶颈不是缺信息,而是有价值的信息在**错误的时间**散落各处,机会来临时取用不到。
解法是把"记住"外包给一个可信的数字系统——第二大脑——让生物大脑专注于思考、连结与创造。
全书的操作核心是 **CODE 四步法**(获取→组织→萃取→表达)配合 **PARA** 组织系统,把日常读到、
想到的一切转化为可复用的知识资产,"打包寄给未来的自己"。核心心法贯穿始终:少而精、按可操作性
组织、为忙碌的未来的你降低取用成本。

## Load-bearing frameworks
1. **CODE 四步法** — Capture 获取 / Organize 组织 / Distill 萃取 / Express 表达。全书主轴,
   信息从摄入到产出的流水线;每一步都是独立、可跨工具的恒久原则。
2. **PARA** — Projects 专案 / Areas 领域 / Resources 资源 / Archives 档案库。按"这条信息对
   哪个专案有用"的**可操作性**归档,而非按主题;以专案为数字档案的主组织单位。
3. **两大超能力** — 第二大脑做两件生物大脑做不好的事:(a) 使构想**具体化**(externalize,
   把模糊念头逼出脑外成为实体);(b) 揭开想法之间**意想不到的连结**。
4. **三大成长阶段** — 记忆 remembering → 连结 connecting → 创造 creating。系统价值随阶段
   递进;判断用户在哪个阶段,决定该强调"存得住"还是"用得出"。
5. **策展人观点 + 引发共鸣** — 获取的筛选心法:像策展人一样掌控信息流,只留**打动你**
   (resonance)的少数精华,而非见到就存。对抗"数字囤积"的核心。
6. **渐进式摘要 (Progressive Summarization)** — 分层加工同一则笔记(原文摘录→加粗→高亮→
   执行摘要),按未来的可用时间选择读第几层,提升"可发现性"而非重读全文。
7. **中间产物封包 (Intermediate Packets)** — 把大专案拆成具体、可复用的小单元(会议笔记、
   研究清单、草图、模板…),循环利用而非每次从零开始。
8. **创意执行三件套** — 发散/收敛的节奏;**构想列岛**(Archipelago of Ideas,先攒素材点再连线);
   **海明威桥**(Hemingway Bridge,在还有动能时收尾并写下"下一步",为下次接力)。
9. **三大维护习惯** — 专案检查表(启动/收尾)、每周与每月检讨、及时整理(just-in-time),
   让系统保持鲜活,像给车做定期保养。
