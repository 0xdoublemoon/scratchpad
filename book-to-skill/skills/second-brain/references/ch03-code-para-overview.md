# ch03 · CODE + PARA 总览
> 从《打造第二大脑》第3章 · 系统如何运作

**核心观点:** 整套方法是两条正交的骨架:CODE 管**信息的流动**(从摄入到产出),PARA 管
**信息的存放**(按可操作性归位)。二者组合覆盖任何职业、工具、笔记平台。

## Frameworks
- **CODE 四步** — Capture 获取(存下引发共鸣的)→ Organize 组织(按可操作性放好)→ Distill
  萃取(找出精髓)→ Express 表达(秀出成果)。是"导航图",不是必须一次走完的线性流程;可在
  不同笔记上处于不同步。
- **PARA 四类** — Projects 专案(有明确终点的当前任务)/ Areas 领域(需长期维持的责任)/
  Resources 资源(未来可能有用的主题参考)/ Archives 档案库(已完成、暂不活跃的)。
- **CODE↔PARA 关系** — Capture 决定存什么;Organize 就是把它放进 PARA;Distill/Express 在
  专案推进时对相关笔记加工调用。

## Techniques
- 向用户解释系统时,先给这张"二维图"再谈细节,避免陷入工具选型。
- 任何一则新信息,先问两问:"它引发共鸣吗?"(要不要存)→"它对哪个专案最有用?"(放哪)。
- 强调步骤**独立**:可以只先大量 Capture,组织与萃取按需再做,别追求一步到位。

## Anti-patterns
- **按主题建复杂层级** — 造"完美"的多级学科文件夹,耗尽精力且日后难检索;应按可操作性而非
  主题组织。
- **把 CODE 当死板流程** — 以为每条笔记都要立刻走完四步;其实多数笔记停在 Capture,需要时才
  往下加工。

## Worked examples
- *读到一篇营销好文* → Capture:只存与手头专案最相关的两三段(不是整篇)→ Organize:丢进对应
  的专案文件夹 → 用到时再 Distill 加粗要点、Express 拼进提案。四步按需展开而非一次做完。
- *"该学心理学"这类念头* → 无对应专案,不建"心理学"大文件夹;归入 Resources 或某个 Area,
  等真有专案(写论文/做分享)时再激活。
