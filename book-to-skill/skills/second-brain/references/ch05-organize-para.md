# ch05 · 组织 PARA — 着眼于可操作性
> 从《打造第二大脑》第5章(含"加值章节·标记系统")

**核心观点:** 别按主题归档,按**可操作性**归档。唯一要问的是"这条信息对我哪个专案最有用",
以专案为主组织单位,信息才会在你真正需要的时刻出现。

## Frameworks
- **PARA** — **Projects 专案**:有明确目标与终点的当前任务(排第一,是主单位);**Areas 领域**:
  需长期维持标准的责任(健康、财务、某个岗位);**Resources 资源**:未来可能有用的主题参考与
  灵感;**Archives 档案库**:已完成/不再活跃的,移出视线但仍可取回。
- **可操作性阶梯** — 四类按"离行动的距离"排序:专案最近、档案库最远。信息随状态在四类间**流动**
  (专案做完→归档;资源被某专案激活→移入专案)。
- **"档案盒"心法(Twyla Tharp)** — 每个专案一个盒子,开盒即产生目的感与责任感;第一件放进去的
  常是写着专案目标的卡片,像船锚把你和初心连住;完成后还能回看、复盘。

## Techniques
- 归档只做一个小动作:把笔记丢进对应**专案文件夹**,或给它打**专案标签**。
- 建专案时先写下 1–2 个目标放进"盒子"最底,迷失时回看。
- 标签作为文件夹的补充:适合跨专案主题或一条笔记属于多处时;别指望靠标签体系一劳永逸。

## Anti-patterns
- **按学科建大文件夹** — 把适应力文章丢进"心理学",主题太宽=不实用;几个月后光是搞清里面有啥
  就要几小时。改问"对哪个专案有用"。
- **追求完美层级** — 花大力气搭多级分类,精力耗在结构上而非要务;PARA 刻意保持扁平。
- **过度设计标签系统** — 一上来定几十个标签规则,维护成本压垮使用;标签宜少、宜按需长出来。

## Worked examples
- *Twyla Tharp 与 Billy Joel 的舞剧* → 开专案先在盒上写剧名(目的感),两张蓝卡写下两个目标
  ("讲好故事""让舞蹈赚钱")垫在盒底当锚,之后一切素材(MV、剪报、越战影像、别的弃案材料)
  都知道往哪放。示范"以专案为单位+目标先行"。
- *读到一篇讲适应力的文章* → 不进"心理学",问"当前哪个专案用得上":没有则入 Resources;有
  (比如在写一篇相关文章)则直接进那个专案文件夹,用时即达。
