# ch04 · 获取 Capture — 储存引发共鸣的信息
> 从《打造第二大脑》第4章

**核心观点:** 获取的目标不是"存得多",而是**存得准**:像策展人一样,只留下真正打动你、日后
用得上的少数精华,把信息流的主导权握在自己手里。

## Frameworks
- **策展人观点 (Curator's Perspective)** — 做信息的评审、编辑与诠释者;网络内容价值分布极不
  均匀,只萃取重中之重,言简意赅写进笔记。获取时越精简,后续组织/萃取/表达越省力。
- **四条获取准则** — (1) 是否带来**启发**?(2) 是否**派得上用场**?(3) 是否与自己**息息相关**?
  (4) 是否**出乎意料**(挑战既有想法)?符合越多越值得存。
- **引发共鸣 (Resonance)** — 最终判据是"这条内容有没有让你心头一动";凭直觉的共鸣,比理性
  清单更可靠。
- **十二道最爱难题(费曼)** — 常备一份 ~12 个开放式长期难题清单,用它筛选新信息:能否为某个
  难题提供新角度?让获取始终服务于你真正好奇的问题。

## Techniques
- 存**片段**不存整篇:书只留精选段落,别把整章塞进笔记。
- 顺手补关键元数据:原文链接、标题、作者/出处、日期(很多获取工具能自动抓)。
- 用"稍后阅读"App 收集,再同步高亮到笔记软件,把摄入与加工分开、降低当下负担。
- 内在产物同样获取:自己的点子、集思广益结果、私下思考也是知识。

## Anti-patterns
- **存太多(数字囤积)** — 见到就存,未来被无关信息淹没,用系统不比上网瞎逛好;宁缺毋滥。
- **无数标签页愧疚症** — 开一堆 tab、因没能保存"一切有趣内容"而焦虑;放下,只取共鸣的。
- **存"专门软件才能开"的大文件** — Photoshop/影音大文件塞进笔记只拖慢性能,应留在专用工具里。

## Worked examples
- *社媒看到一篇心理学文章* → 存进稍后阅读 → 某晚随手读、高亮几段最有趣的 → 同步进笔记,只留
  高亮片段+原文链接。没存全文,后续加工反而更轻。
- *建"十二难题"清单* → 写下如"如何让学校给特殊需求学生更多资源"等长期困惑;此后遇到相关研究
  就自然想存,获取有了方向而非漫无目的。
