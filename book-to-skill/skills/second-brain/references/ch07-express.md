# ch07 · 表达 Express — 秀出你的成果
> 从《打造第二大脑》第7章

**核心观点:** 表达是创造的终点,但不必等一切就绪才分享。把工作拆成可复用的小块、尽早频繁
产出、收集反馈再迭代——专注力太贵,不能让创造过程的"半成品"白白流失。

## Frameworks
- **中间产物封包 (Intermediate Packets, IP)** — 工作中拆出的具体小单元:一组会议笔记、一份
  研究发现清单、集思广益结果、市场分析、通话待办列表…每个都能被更大的专案复用。
- **五类可复用 IP** — 提炼的笔记、他人成果的摘录、工作中的产物、你完成的可交付物、别人的
  可交付物。把报告/图/PPT/文案都当**知识资产**,而非用完即弃。
- **从小处着手 (Think Small)** — 一次只造一个 IP,不担心它够不够完美;创造 = 持续、一点一滴
  交付价值的循环,而非闷头数月的豪赌。

## Techniques
- 拆专案时先问:"这些构件里,哪些我能**复用或组合**而不必从零造?"
- 每完成一个 IP 就存进第二大脑并打好标签,让它可被下个专案检索到。
- 尽早把"半成品"发给老板/客户/同伴求反馈,再进下一块——早反馈=好反馈+少返工。
- 更强的做法:不止刻意造 IP,更要**察觉**日常已生成的 IP 并顺手存下。

## Anti-patterns
- **只崇拜最终成果** — 笔记、草稿、大纲、反馈等中间产物不被珍视→专注力产出的半成品像沙雕被
  海浪冲走,下次又从零开始。
- **憋大招** — 与世隔绝干几周再拿出来,方向早偏;应小步交付、频繁校准。
- **拆了不管** — 只把大任务拆小却没有系统管理这些小块,反而增加追踪负担;拆分必须配 IP 库。

## Worked examples
- *筹办一场大型会议* → 别从零:拆出构件(场地清单、议程模板、讲者名单、宣传文案…),逐一问
  "能否复用/借用现成的",很多构件其实过去就有。把大专案降解为可拼装的 IP。
- *线上课程宣传,只有几周* → 没时间新研究,直接调出"在线教育"夹里的旧笔记,加粗最契合当下的
  部分即用;省下的时间去落实访谈里的新点子。示范 IP 复用带来的杠杆。
